- (event) Propriétaire délègue les bifurcations aux recommandations (« cesses de tergiverser, go ») : mode rapide, rendu visuel oui, dépôt actuel puis dépôt dédié à la 1re release publique, code anglais/UI française, 2 modules, org.buket.app.
- (decision) AD-1 [ADOPTED] Deux modules Gradle : :core (Kotlin pur, zéro dépendance Android — moteurs, modèles, formats) et :app (Android/Compose). Empêche le couplage moteurs↔Android du prototype.
- (decision) AD-2 Code, API et commits en anglais ; textes UI et docs utilisateur en français via resources (i18n N5). Empêche le mélange linguistique incohérent du prototype.
- (decision) AD-3 Source de vérité = fichiers : ticket = paire image + JSON versionné ; tout état d'app (liste, file) est reconstructible par balayage du stockage ; tout cache/DB est jetable. Empêche la double vérité fichiers/base.
- (decision) AD-4 Pipeline unidirectionnel à étapes idempotentes CAPTURED→CROPPED→OCRED→EXTRACTED→DEPOSITED, orchestré par WorkManager, chaque étape lit/écrit le disque privé et est relançable. Empêche pertes sur kill et couplage d'étapes.
- (decision) AD-5 Stockage privé d'abord, dépôt SAF ensuite par copie ; purge de la copie privée uniquement après dépôt confirmé. Empêche toute perte (FR-1.4/5.4) et toute fuite MediaStore (FR-5.6).
- (decision) AD-6 OCR derrière une interface OcrEngine (texte + boîtes) ; implémentation brique 1 : PaddleOCR mobile via ONNX Runtime embarqué. Empêche le verrouillage sur un moteur ; version des modèles fixée au spike S-0.
- (decision) AD-7 Extraction dans :core : portage en anglais des moteurs hérités (LineReassembler, ReceiptAnalyzer) ; chaque champ = valeur + confiance ; règle « champ vide plutôt qu'invention » (FR-3.3).
- (decision) AD-8 UI Compose mono-activité, un ViewModel par écran, navigation par état (3 écrans en brique 1, pas de lib de navigation) ; 100 % des textes dans strings.xml (interdit en dur).
- (decision) AD-9 Jamais d'échec silencieux : tout item de la file a un état visible (dont FAILED + raison + relance) ; toute erreur utilisateur a un message actionnable.
- (decision) AD-10 Tests : :core en JUnit pur obligatoire par story ; corpus de tickets réels (texte OCR anonymisé) versionné dans les ressources de test — le ticket Quick du prototype y entre d'office.
- (decision) AD-11 schema_version entier dans chaque JSON ; lecteurs tolérants : champs inconnus ignorés en lecture et préservés à la réécriture.
- (decision) AD-12 applicationId org.buket.app ; semver ; clé de signature hors git (leçon héritée).
- (direction) Différés : versions exactes des modèles ONNX (spike S-0), algorithme précis de dédoublonnage, son du bip, multi-photos, build reproductible F-Droid, migration vers dépôt dédié buket.
- (event) spine finalized — 12 AD distillés, rendu visuel publié pour le propriétaire, lint indisponible (vérification manuelle : chaque AD porte Binds/Prevents/Rule, différés nommés).
