# Projet « Jardin » — refonte professionnelle sous méthode BMAD 🌱

*Nouveau départ acté le 4 juillet 2026. Identité du produit à définir en phase Analyst.*

Ce dépôt héberge la refonte, **pilotée par la méthode BMAD** (framework
officiel v6, installé dans `_bmad/` et `.claude/skills`), de l'application de
scan de tickets / éducation à la gestion de l'argent prototypée de v1 à
v5.0-beta1.

## État

| PM — PRD | ⬜ |
| Architect — architecture | ⬜ |

## Le prototype archivé

L'application précédente (« Scan Tickets », v1 → v5.0-beta1) est archivée,
fonctionnelle, sur la branche
[`archive/scan-tickets-v5beta`](https://github.com/trygonome/Apks/tree/archive/scan-tickets-v5beta).
Ses acquis réutilisables sont inventoriés dans [`docs/HERITAGE.md`](docs/HERITAGE.md).

La toute première app du dépôt (« Budget Voice ») reste archivée sur
[`archive/budget-voice-v1`](https://github.com/trygonome/Apks/tree/archive/budget-voice-v1).

## Règles du nouveau projet

1. **Aucun code avant le PRD validé** — chaque story exige des critères
   d'acceptation approuvés par le propriétaire du produit avant implémentation.
2. **100 % local** — non négociable, hérité et confirmé.
3. Nouvelle identité applicative (nom, paquet, clé de signature) — décidée en
   phase PM, pas improvisée.
